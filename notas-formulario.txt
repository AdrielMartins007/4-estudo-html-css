- TAGS FORMULARIO -

form - É a tag mais recomendada para ser usada em formularios, alem de ajudar o dev e o navegador a se localizar dentro do codigo, ele tem funções importantes como 'action' e 'autocomplete' por exemplo, que ajuda a melhorar mais ainda a estrutura do codigo, alem de validação de dados como email e senha de forma segura.

action - Tag de ação para envio de dados para uma determinada linguagem de banco de dados ou armazenamento de dados.

label - Identificador de dados para o navegador, onde vc informa o tipo de dado que irá ser acrescentado no input. Tem tambem a função de aumentar a sensibilidade maior do clique no navegador e celular.

input - caixinha onde tem várias funções para entrada e envio de dados como nome, email e botao de envio de dados.

get - metodo onde os dados que foram enviados dentro do formulario ficam visiveis dentro da url do navegador, normalmente ele fica ativo automaticamente.

post - metodo mais seguro os dados 'sensiveis' nao aparecem na url de navegação.

autocomplete - tag que permite que o navegador de sugestoes para o nome ou senha de usuario. Existe uma diferença, se caso for usado dentro do 'form', so tem duas opções o 'on' e o 'off'. Dentro do input vc tem varias opções e ajuda mais ainda o navegador a dar sugestoes de nome e senha, quer seja nova ou antiga.

- TAG DE OBRIGATORIEDADE -

required - Comando que obriga o usuario a digitar os dados para nao enviar nulo, essa tag é incluida dentro do input.

minlength - Comando que limita a quantidade de caracteres dentro do input.

maxlength - comando que maximiza a quantidade de caracteres no input.

- ESTETICA -

size - Determina o tamanho da caixa input.

placeholder - Comando que insere um texto dentro do inputo, uma dica.

fieldset - Tag que envelopa uma parte dos usuarios determinando sua importancia ou função.

legend - Tag para colocar uma legenda, nesse exemplo foi colocado dentro do 'fieldset'.
